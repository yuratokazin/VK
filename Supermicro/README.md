Чтобы изображения из Typora корректно отображались на GitHub, вам необходимо загрузить файлы самих картинок в репозиторий и обновить пути к ним в Markdown-файле.

Вот пошаговая инструкция:

1. Подготовка изображений

В папке на вашем компьютере, где лежит файл `.md`, создайте папку `images`. Скопируйте в неё все изображения, которые вы использовали в документе (их можно найти по пути, указанному в коде, например: `/home/oem/.config/Typora/typora-user-images/`).

2. Загрузка в GitHub

Загрузите папку `images` в ваш репозиторий `VK` по пути `Supermicro/`. Это можно сделать через интерфейс сайта GitHub (кнопка **Add file** -> **Upload files**) или через Git-команды.

3. Изменение путей в коде

В самом тексте руководства на GitHub вам нужно заменить локальные абсолютные пути (начинающиеся с `/home/oem/...`) на относительные.

**Пример замены:**

- **Было:**
  `![image-20250817212750610](/home/oem/.config/Typora/typora-user-images/image-20250817212750610.png)`
- **Стало:**
  `![image-20250817212750610](images/image-20250817212750610.png)`

Как сделать это быстро:

1. Откройте файл в VS Code или любом текстовом редакторе.
2. Нажмите `Ctrl + H` (замена).
3. В поле "Найти" вставьте: `/home/oem/.config/Typora/typora-user-images/`
4. В поле "Заменить на" вставьте: `images/`
5. Нажмите "Заменить всё", сохраните и отправьте изменения на GitHub.

Почему это важно:

GitHub не имеет доступа к файловой системе вашего компьютера. Чтобы показать картинку, он ищет её либо по прямой ссылке в интернете, либо внутри того же репозитория. Использование относительного пути `images/название.png` сообщает GitHub, что файл нужно искать в папке `images`, которая лежит рядом с вашим `.md` файлом.
