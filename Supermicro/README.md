### **Предупреждение по электропитанию**  
**Внимание!** Во избежание повреждения материнской платы и компонентов используйте блок питания с поддержкой следующих разъемов:  
- **24-контактный**  
- **Два 8-контактных**  
- **Один 4-контактный**  

Обязательно подключите:  
- **24-контактный** и **8-контактные** разъемы — для стабильного питания системы.  
- **4-контактный** разъем — опциональный, но Supermicro рекомендует его подключение для оптимального энергоснабжения.  

Если индикаторы ведут себя некорректно (например, не горят или мигают с ошибками), проверьте подключение питания и состояние BMC/IPMI.

### **1-2 Обзор процессора и чипсета**  
Материнская плата **X9DBU-3F/X9DBU-iF**, основанная на процессорах **Intel E5-2400** (сокет B2) и чипсете **C602/C606**, обеспечивает высокую производительность и функциональность для двухпроцессорных платформ.  

#### **Ключевые особенности:**  
- **Технология Intel QuickPath Interconnect (QPI):**  
  - Скорость передачи данных до **8.0 GT/s** (точечное соединение между процессорами).  
- **Функции чипсета C602/C606:**  
  - Поддержка **PCI-Express 2.0** (некоторые модели — **Gen. 3**).  
  - Управление питанием **ACPI 3.0b/4.0**.  
  - Интерфейсы **USB** (задняя и передняя панели).  
  - **Intel Rapid Storage Technology** (оптимизация работы накопителей).  
  - **Intel VT-d** (виртуализация ввода-вывода).  
  - **Intel Trusted Execution Technology** (безопасность).  
  - Поддержка **SPI, DMI, AHCI**.  

---

### **1-3 Специальные функции**  

#### **Восстановление после пропадания питания**  
BIOS позволяет настроить реакцию системы на восстановление электропитания:  
- **Оставаться выключенной** (требуется ручное включение).  
- **Автоматически включиться**.  
- *По умолчанию:* **Last State** (возврат к предыдущему состоянию).  

---

### **1-4 Мониторинг состояния системы**  
Материнская плата оснащена встроенной системой мониторинга:  

#### **Контроль напряжений**  
Постоянно отслеживаются:  
- **1.5V, 1.8V, +3.3V, +5V** (основные).  
- **3.3V Standby, +5V Standby** (дежурные).  
- **Напряжение батарейки CMOS**.  
При отклонениях выдается предупреждение или ошибка.  

#### **Управление вентиляторами**  
- **IPMI-прошивка** регулирует скорость вращения CPU и корпусных вентиляторов.  
- **Термодатчики** активируют охлаждение при превышении заданного порога температуры.  

#### **Контроль температуры**  
- **CPU:** Автоматическое включение вентиляторов при перегреве.  
- **Корпус:** Оповещение о высокой температуре внутри шасси.  
> **Важно:** Для предотвращения перегрева обеспечьте достаточный воздушный поток.  

#### **Оповещения через SuperDoctor**  
- В **Windows/Linux** утилита **SuperDoctor III/II** уведомляет о:  
  - Превышении температуры CPU/системы.  
  - Аномальных напряжениях или скорости вентиляторов.  

---

### **1-5 Функции ACPI**  
**ACPI** (Advanced Configuration and Power Interface) — стандарт управления питанием:  

#### **Основные возможности:**  
- **Автоматическое включение/выключение** периферии (HDD, CD-ROM, сетевые карты).  
- **Поддержка Plug and Play** (динамическая конфигурация устройств).  
- **Совместимость** с Windows 7/Vista/2008.  

#### **Режимы работы:**  
- **S0** (рабочий).  
- **S3** (Suspend to RAM).  
- **S4/S5** (полное выключение).  

---

### **Примечания**  
1. Для настройки **IPMI** используйте [Embedded IPMI Configuration Guide](http://www.supermicro.com/support/manuals/).  
2. Рекомендуется **активное охлаждение** для процессоров с TDP до 95 Вт.  
2. **Максимальный TDP процессора** зависит от системы охлаждения. Проверьте совместимость с корпусом и кулером. 

---



### **1-6 Индикация режима ожидания (Suspend-State LED)**  
- **Медленное мигание** силового светодиода корпуса указывает на переход CPU в **режим ожидания (Suspend)**.  
- При нажатии любой клавиши система **просыпается**, а LED **перестает мигать** и горит постоянно.  

---

### **1-7 Блок питания**  
#### **Ключевые требования:**  
- Обязательные разъемы:  
  - **24-контактный ATX** (основной).  
  - **Два 8-контактных (JPW2/JPW3)** + **4-контактный (JPW4, опционально, но рекомендуется)**.  
- Ток **1.5A** на порты Ethernet.  

#### **Рекомендации:**  
- Используйте **качественный БП** с сертификацией **ATX 2.02+/SSI**.  
- В зонах с нестабильным напряжением:  
  - **Фильтр помех** для защиты от шумов.  
  - **Сетевой фильтр** от скачков напряжения.  

> **Внимание!** Неправильное подключение питания может повредить плату!  

---

### **1-8 Super I/O (Nuvoton W83527)**  
#### **Функционал:**  
- **2 UART-порта** (16550-совместимые) с поддержкой:  
  - Скорость до **1 Мбит/с** (для высокоскоростных модемов).  
  - FIFO-буфер (16 байт), управление модемом.  
- **Управление питанием ACPI** (через SMI/SCI) для снижения энергопотребления.  

---

### **1-9 Расширенное управление питанием**  
#### **Технологии:**  
1. **Intel Intelligent Power Node Manager (IPNM):**  
   - Контроль температуры и энергопотребления в реальном времени.  
   - Требования:  
     - Блок питания с **PMBus 1.1/1.2**.  
     - ПО **NMView** (опционально).  
   - **Зависит от версии ME-прошивки** (1.5 или 2.0).  

2. **Management Engine (ME):**  
   - Встроенный в чипсет **ARC-контроллер** для серверных платформ.  
   - Обеспечивает **Server Platform Services (SPS)**.  

---

### **1-10 IPMI-контроллер (Nuvoton WPCM450R)**  
#### **Возможности:**  
- **Удаленное управление** через **KVM-редирект** (клавиатура/видео/мышь).  
- **Интерфейсы:**  
  - **PCI** — для работы с графикой.  
  - **USB 2.0/1.1** — эмуляция периферии.  
  - **LPC** — управление Super I/O.  
  - **SMBus (x6), PECI, GPIO** — мониторинг компонентов.  
- **Сетевые подключения:**  
  - Через **Ethernet PHY** или **NCSI** (совместное использование).  

---

### **Важные примечания**  
- Для **IPMI** требуется настройка через [Supermicro manuals](http://www.supermicro.com/support/manuals/).  
- **PMBus** обязателен для функций **IPNM**.  

Для оптимальной работы соблюдайте рекомендации по питанию и охлаждению!



### **WPCM450R — Интерфейсы и функции контроллера BMC**

#### **1. Поддержка памяти DDR2**
- **Тип памяти:** 16-бит DDR2  
- **Максимальная частота:** 220 МГц  
- **Топология:** Точечное соединение (point-to-point) для лучшей целостности сигнала.  

---

#### **2. PCI-интерфейс**
- **Спецификация:** PCI 2.3 (32-бит, 33 МГц, 3.3V)  
- **Назначение:** Связь с **PCI-мостом** для работы графического контроллера.  

---

#### **3. Ключевые функции контроллера**  
1. **Управление через IPMI 2.0:**  
   - **Serial over LAN** — удаленный доступ к консоли.  
   - **KVM over LAN** — управление клавиатурой, видео и мышью.  
   - **SNMP-оповещения** (LAN Alerting).  
   - **Журнал событий** (Event Log).  

2. **Интерфейсы расширения:**  
   - **X-Bus** — параллельный интерфейс для I/O.  
   - **SPI Flash** — загрузка BIOS и прошивки.  
   - **RMII** (Reduced Media Independent Interface) — упрощенный Ethernet-интерфейс.  

3. **Мониторинг оборудования:**  
   - Аналоговые и цифровые видеовыходы.

   - **Не зависит от ОС** (работает на уровне железа).  

4. **Безопасность и управление:**  
   - **RMCP+** — защищенный протокол для IPMI.  
   - Поддержка инструментов:  
     - **IPMIView** (графический интерфейс).  
     - **CLI** (командная строка).

### **Важное примечание**  
Для настройки IPMI обратитесь к [IPMI User’s Guide](http://www.supermicro.com/support/manuals/).  

Контроллер **WPCM450R** обеспечивает **полный цикл мониторинга и управления** сервером, включая аварийные оповещения и удаленный доступ, даже без загруженной ОС.



## Глава 2: Установка 

#### **2-1 Стандартные предупреждения**  
Перед установкой компонентов внимательно прочтите все предупреждения. При возникновении вопросов обратитесь в техническую поддержку Supermicro.  

##### **Обращение с батарейкой**  
⚠ **Предупреждение!**  
- **Опасность взрыва** при неправильной замене батарейки.  
- Используйте **только рекомендованный производителем тип** (CR2032 или аналогичный).  
- Утилизируйте старые батарейки **согласно инструкциям производителя**.  

##### **Утилизация оборудования**  
⚠ **Предупреждение!**  
- Утилизация продукта должна производиться **в соответствии с национальными законами**.  

---

#### **2-2 Защита от статического электричества**  
Электростатические разряды (ESD) могут повредить компоненты.  

##### **Меры предосторожности:**  
- Используйте **антистатический браслет**.  
- Перед извлечением платы **коснитесь заземленного металла**.  
- Держите плату **только за края** — не касайтесь чипов, контактов и разъемов.  
- Не трогайте **контакты модулей памяти и процессоров**.  
- Храните плату в **антистатической упаковке**, когда она не используется.  
- Убедитесь, что корпус обеспечивает **заземление** (между БП, шасси и материнской платой).  

##### **Распаковка**  
- Плата поставляется в **антистатической упаковке**.  
- Перед распаковкой **обеспечьте защиту от статики**.  

---

### **Ключевые моменты**  
1. **Батарейка CMOS** — замена только на аналогичную (CR2032).  
2. **Антистатика** — обязательна при работе с электроникой.  
3. **Утилизация** — соблюдайте экологические нормы.  

Для дальнейших шагов по установке обратитесь к руководству.

### **Особенности работы с разными типами памяти**

- **Частоты:** 1066/1333 МГц (зависит от напряжения 1.35V/1.5V)  
- **Ограничения:**

#### **RDIMM (ECC)**
- **Поддерживаемые объемы:** 1GB–32GB

- **Поддерживаемые объемы:** 16GB, 32GB  
- **Частоты:** 1066/1333 МГц (1.5V)

1. **Не смешивайте разные типы памяти** (RDIMM, UDIMM, LRDIMM) в одной системе.  
2. **Все модули в системе должны иметь одинаковую частоту** — работа будет на частоте самого медленного модуля.  
3. **Не устанавливайте одновременно ECC и Non-ECC** модули.

### **Функционал UID (Unit Identifier) на материнской плате X9DBU-3F/X9DBU-iF**

#### **1. Назначение UID**
- **Визуальная идентификация** сервера в стойке (особенно полезно в дата-центрах)
- **Упрощение обслуживания** - помогает быстро найти нужный сервер среди множества

#### **2. Элементы управления**
- **UID Switch (кнопка)**:
  - Расположена на задней панели ввода-вывода
  - Физическая кнопка для ручного управления индикацией

#### **3. Индикация**
- **Два светодиода** (синий/голубой):
  1. **Задний UID LED** (на задней панели)
  2. **Передний UID LED** (подключается к разъему на плате)

#### **4. Принцип работы**
1. **Однократное нажатие** кнопки UID:
   - Активирует **оба светодиода** (20-19 контакты на разъеме FP)
   - Сервер помечается для обслуживания

2. **Повторное нажатие**:
   - Отключает светодиодную индикацию
   - Система возвращается в нормальный режим

#### **5. Разъем для фронтального индикатора (20-19 контакты)**
| Контакт | Назначение | Цвет кабеля |
| ------- | ---------- | ----------- |
| 19      | Анод (+)   | Синий       |
| 20      | Катод (-)  | Белый       |

#### **6. Важные особенности**
- Индикация работает **независимо от состояния системы** (питание может быть выключено)
- Поддерживает **удаленное управление** через IPMI
- Совместима с **корпусными индикаторами** стандартных серверных шасси

#### **7. Рекомендации по использованию**
1. При обслуживании сервера **включите UID** для быстрой идентификации
2. Для подключения фронтального светодиода используйте **двухконтактный кабель 1.8mm pitch**
3. Не превышайте **ток 20mA** через светодиод

> **Примечание:** UID также активирует подсветку кнопки питания в некоторых корпусах Supermicro.



### **UID Switch: Распиновка и подключение**

#### **Назначение контактов**

| Pin # | Назначение | Описание                       | Цвет провода (типовой) |
| ----- | ---------- | ------------------------------ | ---------------------- |
| 1     | Ground     | Общий провод                   | Черный                 |
| 2     | Ground     | Дублирующий общий провод       | Черный                 |
| 3     | Button In  | Сигнальный вход кнопки (+3.3V) | Синий/Красный          |
| 4     | Ground     | Дополнительный общий провод    | Черный                 |

#### **Особенности подключения**
1. **Трехпроводная схема** (несмотря на 4 контакта):
   - Контакты 1, 2 и 4 соединены вместе (общий Ground)
   - Контакт 3 - сигнальный

2. **Логика работы**:
   - В нормальном состоянии: контакт 3 подтянут к +3.3V через резистор
   - При нажатии: замыкается на Ground (активный низкий уровень)

3. **Рекомендации**:
   - Используйте кабель с шагом 2.54mm (стандартный DuPont)
   - Максимальный ток через контакт 3: 5mA
   - Длина кабеля не должна превышать 50см

#### **Типовая схема подключения**
```
[Кнопка UID] 
   │
   ├───[Pin 3] Button In (сигнал)
   │
   └───[Pins 1,2,4] Ground
```

#### **Диагностика неисправностей**
1. **Кнопка не работает**:
   - Проверить целостность кабеля
   - Измерить +3.3V на Pin 3 (относительно Ground)

2. **Ложные срабатывания**:
   - Проверить на короткое замыкание
   - Убедиться в отсутствии перекрестных помех

> **Важно!** UID switch работает независимо от состояния питания системы (через BMC). Для полной функциональности требуется подключение разъема JPW1 (основное питание).



### **UID LED (LE2) Status Indicators**

#### **Индикация состояния сервера через UID светодиод**

| **Цвет/Состояние** | **Операционная система** | **Статус системы**     |
| ------------------ | ------------------------ | ---------------------- |
| **Синий: горит**   | Windows OS               | Сервер идентифицирован |
| **Синий: мигает**  | Linux OS                 | Сервер идентифицирован |

#### **Особенности работы:**
1. **Windows ОС**:
   - Постоянная синяя подсветка
   - Активируется:
     * При физическом нажатии UID-кнопки
     * Через IPMI-команды
     * Через утилиту Supermicro (Supermicro System Manager)

2. **Linux ОС**:
   - Мигающий синий индикатор (1 Гц)
   - Особенности реализации:
     * Частота мигания может меняться через BMC
     * Поддержка в стандартных драйверах ipmitool

3. **Общая логика**:
   - Индикация сохраняется после перезагрузки
   - Работает даже при выключенном основном питании (за счет BMC)
   - Совместима с:
     * Redfish API
     * IPMI 2.0 стандартными командами

#### **Управление через IPMI:**
```bash
# Включить индикацию
ipmitool raw 0x3c 0x0a 0x01

# Выключить индикацию 
ipmitool raw 0x3c 0x0a 0x00

# Проверить статус
ipmitool raw 0x3c 0x0b
```

#### **Типовые сценарии использования:**
1. **Идентификация в стойке**:
   - Техник находит нужный сервер по синему светодиоду
   - После обслуживания отключает индикацию

2. **Автоматизированный мониторинг**:
   - Система управления включает индикацию при обнаружении ошибок
   - Мигающий режим указывает на ожидание обслуживания

3. **Интеграция с DCIM**:
   - Соответствие стандарту IEC 62343-5-1
   - Поддержка протоколов SNMP trap для событий UID

> **Примечание:** В некоторых версиях BIOS можно настроить чувствительность кнопки и поведение светодиода через раздел "BMC Configuration".

### **Функционал системы UID (Unit Identifier)**

#### **Управление двойной LED-индикацией**
1. **Синхронная активация**:
   - Однократное нажатие **физической кнопки UID** включает:
     - **Задний UID-светодиод** (на задней панели)
     - **Передний UID-индикатор** (подключен через контакты 19-20)
   - Повторное нажатие отключает оба индикатора

2. **Характеристики индикаторов**:
   | Расположение    | Цвет  | Ток  | Управление                       |
   | --------------- | ----- | ---- | -------------------------------- |
   | Задняя панель   | Синий | 15мА | Прямое управление BMC            |
   | Передняя панель | Синий | 10мА | Через разъем FP (контакты 19-20) |

#### **Дистанционное управление через IPMI**
1. **Командная строка**:
   ```bash
   # Активировать UID
   ipmitool raw 0x3C 0x0A 0x01 0x00
   
   # Деактивировать UID
   ipmitool raw 0x3C 0x0A 0x00 0x00
   
   # Проверить статус
   ipmitool raw 0x3C 0x0B
   ```

2. **Веб-интерфейс**:
   - Вход в BMC → Server Health → UID Control

#### **Электрические параметры**
1. **Подключение переднего индикатора**:
   ```
   Контакт 19: +3.3V (анод)
   Контакт 20: Земля (катод)
   ```
   - Максимальный ток: 20мА
   - Поддержка PWM-диммирования (через прошивку BMC)

2. **Цепь кнопки**:
   ```
   [Кнопка UID]
     ├── Подтяжка к +3.3V (внутри BMC)
     └── Замыкание на землю при нажатии
   ```

#### **Диагностика неисправностей**
1. **Индикаторы не горят**:
   - Проверить наличие +3.3V дежурного питания
   - Убедиться в версии прошивки BMC ≥ 1.30
   - Прозвонить цепь контактов 19-20

2. **Кнопка не работает**:
   - Измерить напряжение на контактах (должно быть 3.3V в разомкнутом состоянии)
   - Проверить механическую целостность кнопки

#### **Рекомендации по эксплуатации**
1. **Для дата-центров**:
   - Ресурс кнопки: 200,000 нажатий
   - Видимость индикатора: до 30 метров в рядах стоек

2. **Интеграция**:
   - Генерация SNMP-трапов при активации
   - Поддержка Redfish API для автоматизации

> **Примечание:** Состояние UID сохраняется после перезагрузки и не сбрасывается при рестарте BMC. Подробности в "Руководстве по прошивке BMC" на [Supermicro Documentation](https://www.supermicro.com/support/manuals/).

### **Разъем передней панели (JF1) - Кнопка NMI**

#### **Распиновка и подключение**
| Контакт | Назначение | Тип сигнала     | Цвет провода (типовой) |
| ------- | ---------- | --------------- | ---------------------- |
| **19**  | NMI_SW#    | Активный низкий | Красный/Желтый         |
| **20**  | Ground     | Земля           | Черный                 |

#### **Функциональные особенности**
1. **Назначение кнопки**:
   - Генерирует **немаскируемое прерывание** (Non-Maskable Interrupt)
   - Принудительно переводит систему в режим отладки
   - Используется для:
     - Аварийного дампа памяти (crash dump)
     - Диагностики зависших систем
     - Активации отладочных режимов

2. **Электрические параметры**:
   - Напряжение логики: 3.3V
   - Ток срабатывания: <5mA
   - Защита от дребезга: встроенная в BMC

#### **Рекомендации по подключению**
1. **Схема подключения**:
   ```
   [Кнопка NMI]
     ├── Контакт 19 (NMI_SW#)
     └── Контакт 20 (GND)
   ```

2. **Требования к кнопке**:
   - Тактовая кнопка без фиксации
   - Нормально-разомкнутый контакт
   - Ресурс: ≥100,000 нажатий

#### **Меры предосторожности**
1. **Защита от случайного нажатия**:
   - Рекомендуется использовать углубленную кнопку
   - Возможна блокировка через BIOS:
     ```bash
     # В настройках BMC:
     ipmitool raw 0x3C 0x0C 0x01 0x00
     ```

2. **Диагностика**:
   - Проверка работоспособности:
     ```bash
     # Через IPMI:
     ipmitool chassis status | grep NMI
     ```

> **Важно!** Для систем с критической нагрузкой рекомендуется отключать физический доступ к NMI-кнопке. Альтернатива - активация NMI только через IPMI.

### **Управление перемычками CMOS и Watch Dog**

#### **1. Сброс CMOS (JBT1)**
**Расположение:** Контактные площадки возле батарейки CMOS  
**Процедура сброса:**

1. Полностью выключите сервер и отсоедините кабель питания
2. Извлеките батарейку CMOS (CR2032)
3. Замкните металлическим предметом (отвертка) оба контакта на 10 секунд:
   ``` 
   [PAD1] ===(замыкание)===[PAD2]
   ```
4. Верните батарейку и включите питание

**Последствия сброса:**
- Сброс BIOS к заводским настройкам
- Удаление всех паролей (включая BMC)
- Сброс настроек RAID и Boot Order

---

#### **2. Конфигурация Watch Dog (JWD1)**
**Расположение:** Рядом с разъемом TPM  
**Режимы работы:**

| Положение | Функция                      | Применение                 |
| --------- | ---------------------------- | -------------------------- |
| **1-2**   | Hard Reset (по умолчанию)    | Перезагрузка при зависании |
| **2-3**   | NMI (Non-Maskable Interrupt) | Прерывание для диагностики |
| **Open**  | Отключено                    | -                          |

**Настройка через BIOS:**
```
Advanced → Watch Dog Configuration → 
- Timeout: [1-255 мин]
- Action: [Reset/NMI]
```

**Пример использования:**
```bash
# Активация через IPMI (60 сек таймаут):
ipmitool raw 0x06 0x24 0x00 0x00 0x00 0x3C
```

---

#### **Сравнение функций**
| Особенность        | JBT1 (CMOS Clear) | JWD1 (Watch Dog)       |
| ------------------ | ----------------- | ---------------------- |
| Физический доступ  | Требуется         | Не обязателен          |
| Влияние на систему | Полный сброс      | Контролируемый рестарт |
| Интеграция         | Только аппаратная | BIOS/IPMI/ОС           |

---

#### **Критические замечания**
1. Для **CMOS Clear**:
   - Не замыкайте контакты при включенном питании!
   - После сброса потребуется:
     ```bash
     # Восстановить настройки BMC:
     ipmitool lan set 1 ipsrc static
     ```

2. Для **Watch Dog**:
   - При NMI-режиме требуется драйвер в ОС
   - Логи событий сохраняются в:
     ```
     /var/log/ipmi.log (Linux)
     Event Viewer → System (Windows)
     ```

> **Важно!** Для промышленных решений с Watch Dog:
> - Используйте внешний мониторинг через `ipmitool sel list`
> - Настройте дублирование сигнала на релейный выход

### Светодиод питания на плате

**Onboard Power LED** 

Светодиод питания на плате расположен на материнской плате (LED2). Когда этот светодиод горит, система включена. Перед извлечением или установкой компонентов обязательно выключите систему и отсоедините шнур питания. Дополнительную информацию см. в таблицах справа.

### **Индикатор питания на материнской плате (LED1)**

#### **Настройки индикации**
| Цвет/Состояние                  | Режим работы системы             | Описание                                                     |
| ------------------------------- | -------------------------------- | ------------------------------------------------------------ |
| **Выключен**                    | Система отключена                | Кабель питания не подключен или отсутствует дежурное напряжение |
| **Зеленый (постоянно)**         | Система включена                 | Основное питание подано, система работает в нормальном режиме |
| **Зеленый (частое мигание)**    | Режим ACPI S1 (Power On Suspend) | Процессор остановлен, оперативная память и устройства остаются под напряжением |
| **Зеленый (медленное мигание)** | Режим ACPI S3 (Suspend to RAM)   | Состояние сохранено в RAM, большинство компонентов отключено |

#### **Технические детали**
1. **Управление:**
   - Контролируется чипсетом Intel C602/C606
   - Источник питания: +5VSB (дежурное напряжение)

2. **Частота мигания:**
   - Быстрое: 2 Гц (S1)
   - Медленное: 0.5 Гц (S3)

3. **Подключение:**
   ```
   LE1:
     Анод (+) → 5VSB (красный провод)
     Катод (-) → Сигнал управления (зеленый провод)
   ```

#### **Рекомендации**
1. **При диагностике:**
   - Если индикатор не горит при подключенном питании:
     ```bash
     # Проверьте дежурное напряжение:
     ipmitool sensor list | grep "5VSB"
     ```
   - Постоянное горение в S3 режиме указывает на ошибку ACPI

2. **Для корректной работы:**
   - В BIOS должны быть включены:
     ```
     Advanced → ACPI Settings → Suspend Mode [Auto]
     ```

3. **Особенности состояний:**
   - **S1:** Быстрый выход (1-2 сек), повышенное энергопотребление
   - **S3:** Медленный выход (5-10 сек), экономия энергии

> **Важно!** При обслуживании системы всегда дожидайтесь полного выключения индикатора перед отсоединением кабеля питания. Мигающие режимы указывают на активное состояние низкого энергопотребления.


> **Важно!** Чипсет C606 поддерживает гибридные конфигурации (SSD+HDD), но не рекомендуется объединять потребительские SATA-диски с корпоративными SAS-накопителями в одном RAID-массиве из-за разницы в производительности. Для максимальной производительности SAS используйте сертифицированные Supermicro диски.



### 2-10 Подключение Serial ATA и SAS

#### **1. Порты Serial ATA (SATA)**
**Конфигурация портов:**
| Порты     | Версия SATA | Макс. скорость | Режимы работы       |
| --------- | ----------- | -------------- | ------------------- |
| I-SATA0/1 | SATA III    | 6 Гбит/с       | AHCI, RAID 0/1/5/10 |
| I-SATA2-5 | SATA II     | 3 Гбит/с       | Legacy IDE, AHCI    |

**Ключевые особенности:**
- Поддержка горячей замены (требуется настройка в BIOS)
- Совместимость с SSD и HDD
- Intel Rapid Storage Technology для RAID-массивов

**Распиновка SATA-портов:**
```
1: Ground    2: TX+      3: TX-
4: Ground    5: RX-      6: RX+
7: Ground
```

#### **2. Порты SAS (только X9DBU-3F/X9DBU-iF)**
**Конфигурация портов:**
| Группа портов | Тип     | Скорость | Особенности              |
| ------------- | ------- | -------- | ------------------------ |
| SSATA/SAS0-3  | SAS 2.0 | 6 Гбит/с | Совместимость с SATA III |
| SAS4-7        | SAS 2.0 | 6 Гбит/с | Полноценные SAS-порты    |

**Функционал:**
- Поддержка RAID 0/1/10 через Intel C606
- T10 PI (защита данных) для корпоративных дисков
- STP (Serial Tunneling Protocol) для SATA-устройств

#### **Рекомендации по подключению**
1. **Для SATA:**
   ```bash
   # Проверка скорости в Linux:
   for i in {0..5}; do cat /sys/class/ata_link/link$i/sata_spd; done
   ```
   - SATA III (6 Гбит/с) идеальны для SSD
   - SATA II (3 Гбит/с) подходят для HDD и оптических приводов

2. **Для SAS:**
   - Используйте сертифицированные Supermicro диски
   - Приоритет для критичных нагрузок:
     ```bash
     # Мониторинг через ipmitool:
     ipmitool sensor list | grep -i "sas_temp"
     ```

3. **Настройка RAID:**
   - Требуется активация в BIOS:
     ```
     Advanced → SATA Configuration → RAID Mode
     ```
   - Документация по HostRAID: [Intel SATA HostRAID Guide](https://www.supermicro.com/support/manuals/)

#### **Сравнение интерфейсов**
| Характеристика | SATA III          | SAS 2.0  |
| -------------- | ----------------- | -------- |
| Макс. скорость | 6 Гбит/с          | 6 Гбит/с |
| Поддержка SSD  | Да                | Да       |
| Горячая замена | Да (с настройкой) | Да       |
| Длина кабеля   | до 1 м            | до 10 м  |

> **Важно!** Для SAS-подключений используйте кабели с экранированием. Смешивание SATA и SAS в одном RAID-массиве не рекомендуется.



### **Распиновка SATA/SAS-портов**

#### **Назначение контактов**
| Контакт | Обозначение | Тип сигнала              | Описание                       |
| ------- | ----------- | ------------------------ | ------------------------------ |
| **1**   | Ground      | Земля                    | Общий проводник                |
| **2**   | TX_P        | Передача (положительный) | Дифференциальная пара передачи |
| **3**   | TX_N        | Передача (отрицательный) |                                |
| **4**   | Ground      | Земля                    | Экранирование                  |
| **5**   | RX_N        | Прием (отрицательный)    | Дифференциальная пара приема   |
| **6**   | RX_P        | Прием (положительный)    |                                |
| **7**   | Ground      | Земля                    | Дополнительное экранирование   |

#### **Технические характеристики**
1. **Скорость передачи:**
   - **SATA III:** 6 Гбит/с (600 МБ/с)
   - **SAS 2.0:** 6 Гбит/с (совместим с SATA III)

2. **Дифференциальные пары:**
   - Импеданс: 100 Ом (±10%)
   - Напряжение: 400-600 мВ (пиковое)

3. **Электрические требования:**
   - Допустимое отклонение напряжения: ±5%
   - Максимальный ток: 1.5A на порт

#### **Схема подключения**
```plaintext
[Устройство (HDD/SSD)]
├── 2 (TX_P) → Красный провод
├── 3 (TX_N) → Белый провод
├── 5 (RX_N) → Синий провод
└── 6 (RX_P) → Желтый провод

[Заземление]
├── 1,4,7 → Черные провода
```

#### **Рекомендации по эксплуатации**
1. **Для стабильной работы:**
   - Используйте кабели длиной ≤1 м (SATA) / ≤10 м (SAS)
   - Избегайте перегибов кабелей (минимальный радиус 30 мм)

2. **Диагностика проблем:**
   ```bash
   # Проверка линка в Linux:
   dmesg | grep -i "sata link"
   ```
   - Ошибка "PHY RDY change" указывает на проблемы с кабелем

3. **При подключении SAS:**
   - Убедитесь в поддержке SSP (Serial SCSI Protocol)
   - Для SATA-устройств активируйте STP в BIOS:
     ```
     Advanced → SAS Configuration → STP Support
     ```

> **Важно!** Не подключайте питание к сигнальным контактам. Для горячей замены SAS-устройств используйте специальные backplane-разъемы с поддержкой SES-2.

# Глава 3: Диагностика и устранение неисправностей

Chapter 3: Troubleshooting

## Процедуры диагностики

Используйте следующие процедуры для устранения неисправностей. Если после выполнения всех шагов проблема не решена, обратитесь к разделам "Процедуры технической поддержки" и/или "Возврат оборудования на обслуживание".

**Внимание:** Всегда отключайте кабель питания перед установкой, заменой или изменением любых аппаратных компонентов.

### Перед включением питания
1. Убедитесь в отсутствии короткого замыкания между материнской платой и корпусом.
2. Отсоедините все ленточные кабели от материнской платы, включая кабели клавиатуры и мыши.
3. Извлеките все дополнительные карты.
4. Установите сначала процессор CPU1 (убедитесь в правильной установке) и подключите разъемы передней панели.

### Нет питания
1. Проверьте отсутствие короткого замыкания между платой и корпусом.
2. Убедитесь в правильном подключении разъемов ATX питания.
3. Проверьте положение переключателя 115V/230V на блоке питания (если есть).
4. Проверьте систему, включая и выключая питание.

5. Возможно разряжена батарейка материнской платы. Проверьте напряжение (~3V). При необходимости замените.

### Нет видеосигнала
1. При включенном питании, но отсутствии изображения, извлеките все дополнительные карты и кабели.
2. Используйте спикер для определения звуковых кодов ошибок (см. Приложение A).

### Сбой загрузки системы
Если система не проходит POST или не реагирует после включения:
1. Проверьте звуковые сигналы:
   - Нет сигнала: попробуйте запустить без модулей памяти, затем только с CPU1
   - Есть сигналы: сбросьте CMOS (раздел 2-8)
2. Извлеките все компоненты, особенно модули памяти, и проверьте звуковые сигналы.
3. Запустите систему с одним модулем памяти для диагностики.

### Потеря настроек BIOS
1. Используйте качественный блок питания.
2. Проверьте/замените батарейку CMOS.
3. Если проблема не решена, обратитесь к поставщику.

### Ошибки памяти
При звуковых сигналах о проблемах с памятью:
1. Убедитесь в совместимости и правильной установке модулей.
2. Используйте модули одинакового типа и скорости.
3. Проверьте тип памяти (RDIMM/LRDIMM/UDIMM).
4. Проверьте модули и слоты, переставляя модули.
5. Убедитесь в полной установке модулей (раздел 2-4).
6. Следуйте таблицам размещения модулей (раздел 2-4).

### Нестабильная работа системы

#### A. Во время или после установки ОС
1. Совместимость CPU/BIOS: проверьте поддержку процессора и обновите BIOS.
2. Память: протестируйте модули (memtest86).
3. HDD: проверьте работоспособность дисков.
4. Охлаждение: проверьте вентиляторы и температуру в BIOS.
5. Питание: убедитесь в достаточной мощности БП.
6. Драйверы: используйте правильные версии.

#### B. До или во время установки ОС
1. Источник установки: проверьте загрузочные устройства.
2. Кабели: проверьте все соединения.
3. Минимальная конфигурация: запустите систему с минимальным набором компонентов.
4. Поиск неисправных компонентов: тестируйте компоненты по отдельности.
5. Заменяйте компоненты по одному для точной диагностики.
6. Проверяйте компоненты в другой системе.

# Глава 4: Настройка AMI BIOS

 Chapter 4: AMI BIOS

## 4-1 Введение

 4-1 Introduction

В этой главе описывается утилита настройки AMI BIOS для материнских плат X9DBU-3F/X9DBU-iF, а также приводятся инструкции по работе с интерфейсом BIOS.

### Основные сведения
- BIOS хранится во флеш-памяти EEPROM и может быть обновлен
- Для входа в настройки BIOS нажмите **<Del>** во время загрузки системы
- В некоторых моделях могут использоваться другие клавиши (<F3>, <F4> и т.д.)

### Структура интерфейса
Интерфейс BIOS состоит из двух основных областей:
1. **Левая часть** - список доступных для настройки параметров
   - Серые пункты - недоступны для изменения
   - Синие пункты - доступны для настройки
2. **Правая часть** - справочная информация и подсказки по управлению

### Управление в BIOS
Используются следующие "горячие клавиши":
- **<F3>** - загрузка оптимальных настроек по умолчанию
- **<F4>** - сохранение изменений и выход
- **<Enter>** - выбор/вход в подменю
- **<ESC>** - выход/отмена
- Клавиши-стрелки - навигация

**Важно!** 
- Обновляйте BIOS только при наличии проблем, связанных с BIOS
- Не прерывайте процесс обновления BIOS - это может привести к необратимым повреждениям системы
- Для восстановления BIOS используйте руководство по восстановлению UEFI BIOS на сайте Supermicro

## 4-2 Главное меню настроек

 4-2 Main Setup

Главный экран BIOS отображает основную информацию о системе и является отправной точкой для всех настроек.

### Основные разделы:
1. **System Information** - информация о системе:
   - Версия BIOS
   - Процессор
   - Объем памяти
   - Системная дата/время

2. **Advanced** - расширенные настройки:
   - Конфигурация процессора
   - Настройки чипсета
   - Управление питанием
   - Параметры PCI

3. **Boot** - настройки загрузки:
   - Приоритет загрузочных устройств
   - Параметры быстрой загрузки
   - Безопасная загрузка

4. **Security** - параметры безопасности:
   - Установка паролей
   - Настройки TPM
   - Безопасность ввода/вывода

5. **Save & Exit** - сохранение настроек:
   - Сохранение изменений
   - Выход без сохранения
   - Загрузка настроек по умолчанию

**Примечание:** Параметры, выделенные **жирным шрифтом**, являются значениями по умолчанию.

### **Функции загрузки**  

#### **Тихая загрузка (Quiet Boot)**  
Управление отображением экрана POST:  
- **Вкл.**: Показывает логотип производителя вместо сообщений POST  
- **Выкл.**: Отображает подробные коды POST и подсчет памяти  

#### **Режим отображения AddOn ROM**  
Определяет, как опциональные ROM (например, RAID-контроллеры) отображают свою инициализацию:  
- **Force BIOS**: Использует настройки системного BIOS  
- **Keep Current**: Сохраняет родной режим отображения адаптера  

#### **Num-Lock при загрузке**  
Устанавливает состояние клавиши NumLock при включении:  
- **Вкл.**: Активирует цифровую клавиатуру  
- **Выкл.**: Деактивирует цифровую клавиатуру  

#### **Ожидание 'F1' при ошибке**  
Управление обработкой ошибок во время POST:  
- **Вкл.**: Система останавливается при ошибках, требуя нажатия F1  
- **Выкл.**: Пытается продолжить загрузку несмотря на ошибки  

#### **Захват прерывания 19 (Interrupt 19 Capture)**  
Управление распознаванием загрузочных устройств:  
- **Вкл.**: Позволяет дополнительным контроллерам загружаться с подключенных дисков  
- **Выкл.**: Ограничивает загрузку только контроллерами материнской платы  

#### **Повторная попытка загрузки (Re-try Boot)**  
- **Вкл.**: Автоматически повторяет попытку загрузки после сбоя  
- **Выкл.**: Останавливается после первой неудачной попытки

### Приложение A  
#### Коды звуковых сигналов BIOS  

Во время выполнения POST (самотестирования при включении), которое проводится при каждой загрузке системы, могут возникать ошибки.  

- **Нефатальные ошибки** — в большинстве случаев позволяют системе продолжить загрузку. Сообщения об этих ошибках обычно отображаются на экране.  
- **Фатальные ошибки** — не позволяют системе завершить процесс загрузки. При возникновении фатальной ошибки следует обратиться к производителю системы для возможного ремонта.  

Фатальные ошибки обычно передаются через серию звуковых сигналов. Номера в списке соответствуют количеству сигналов для каждой конкретной ошибки.  

### A-1 Коды звуковых сигналов BIOS  

| Количество сигналов / Светодиод | Сообщение об ошибке          | Описание                                                     |
| ------------------------------- | ---------------------------- | ------------------------------------------------------------ |
| 1 сигнал                        | **Refresh**                  | Готово к загрузке                                            |
| 5 коротких + 1 длинный сигнал   | **Memory error**             | Память не обнаружена в системе                               |
| 5 сигналов                      | **No Con-In или No Con-Out** | **Con-In**: USB- или PS/2-клавиатура, PCI или Serial Console Redirection, IPMI KVM/SOL. **Con-Out**: Видеоконтроллер, PCI или Serial Console Redirection, IPMI SOL |

На странице загрузки выберите:  
✔ Подходящую версию (32-bit или 64-bit)  
✔ Совместимую с вашей версией Linux (RHEL, CentOS, Ubuntu и др.)  

#### **3. Установка**  
##### **Для RPM-дистрибутивов (RHEL/CentOS):**  
```bash
wget [URL_скачивания_пакета.rpm]
sudo rpm -ivh SuperDoctorII-*.rpm
```

##### **Для DEB-дистрибутивов (Ubuntu/Debian):**  
```bash
wget [URL_скачивания_пакета.deb]
sudo dpkg -i SuperDoctorII-*.deb
```

#### **4. Запуск и настройка**  
После установки:  
```bash
sudo /opt/SuperDoctorII/sdii-gui  # Графический интерфейс  
sudo /opt/SuperDoctorII/sdii-cmd  # Консольная версия  
```

#### **5. Автозапуск (опционально)**  
Чтобы SuperDoctor II запускался автоматически:  
```bash
sudo systemctl enable sdii-service  # (если доступен)  
```
